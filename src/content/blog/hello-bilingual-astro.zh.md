# 你好，欢迎光临

这是我新个人站点上的第一篇文章，所以它既是一块欢迎垫，也是一篇"版本说明"——简单交代一下这个地方是什么、怎么搭起来的，以及我打算往里放些什么。

我叫 Kun，目前在利兹大学（University of Leeds）读高级计算机科学硕士。大部分时间花在自然语言处理（NLP）和大语言模型上，外加一个戒不掉的副业爱好：机器学习竞赛。过去很长一段时间，我的文字散落在各个仓库的 README 和竞赛 notebook 里。这个站点，是我想把其中还算像样的部分收拢到一处、慢下来、把事情真正讲清楚的一次尝试。

## 一个双语站点

这里的每篇文章都有两种语言版本：English 和简体中文。它们不是机器翻译——两个版本我都各自写一遍，尽量让每一种读起来都自然。有时候中文版的说法会和英文版略有出入，我觉得这没什么不好。两种语言是把同一个想法想两遍的两条路径；两边都写一遍，往往会暴露出那些我"自以为"懂了、其实没懂的地方。

如果你进来时看到的是另一种语言，每一页都有切换的开关。

## 它是怎么搭起来的

站点用 [Astro](https://astro.build/) 构建，部署在 GitHub Pages 上。选这条路，大致有几个原因：

- **内容优先。** 文章就是普通的 Markdown，加一点 YAML frontmatter。我宁愿在一个段落上较劲，也不想在构建配置上耗神。
- **快而轻。** Astro 默认几乎不向页面塞 JavaScript，所以加载很快，读起来也清爽。
- **托管免费又省心。** 用 GitHub Pages，一次 `git push` 就是全部的部署步骤，没有需要时刻照看的服务器。

它有意做得很朴素。重点是文字本身，而不是框架。

## 你可以期待什么

大致是三条线，对应我真正在花时间做的事：

- **NLP 与大模型。** 来自课程和业余项目的笔记——比如法律信息检索、模型推理挑战，以及从文档里抽取结构化数据。这里更多会是"我试了什么、又在哪儿翻了车"，而不是什么宏大的断言。
- **机器学习竞赛。** 我把 Kaggle 当成一间实验室。我折腾过的题目挺杂——鸟鸣声分类、ARC-AGI 推理基准、F1 进站预测、车辆检测、恒星天体分类等等。这些大多以公开实验记录的形式放在我的 [GitHub](https://github.com/Beiciccc) 上，我更想把其中的经验教训写出来，而不只是晒一晒排行榜。
- **在利兹读书。** 偶尔写写关于这个硕士本身的感受——哪些地方豁然开朗、哪些还卡着，以及一节条理分明的课和一份乱糟糟的真实数据集之间的落差。

关于"诚实"的一点说明：我会写下各种尝试、走过的死路，以及那些只成功了一半的东西。如果某件事确实做得不错，我会照实说；但我不打算把结果吹得好看。反正真正有意思的，通常都是过程本身。
